# 故障排除指南

## 常见问题：页面文件太小 (OS Error 1455)

### 问题描述
```
页面文件太小，无法完成操作。 (os error 1455)
```

这是Windows系统常见的虚拟内存不足错误，通常发生在加载大型机器学习模型时。

---

### ✅ 解决方案 1：使用低内存模式启动（推荐）

我们已为你创建了专门的低内存启动模式：

1. 双击 `start_server.bat`
2. 选择 **[3] 低内存模式**
3. 在前端界面：
   - 选择模型：**ViT-GPT2 图像描述（轻量、稳定）**
   - 选择设备：**CPU**

这个模式会：
- 限制PyTorch线程数为2
- 启用CUDA内存碎片优化
- 默认使用轻量模型
- 强制使用CPU避免GPU显存溢出

---

### ✅ 解决方案 2：增加Windows虚拟内存（页面文件）

如果方案1仍不够，请手动增加Windows虚拟内存：

#### Windows 10/11:

1. **右键点击**「此电脑」→ **属性**
2. 点击 **「高级系统设置」**
3. 在 **「性能」** 区域，点击 **「设置」**
4. 切换到 **「高级」** 选项卡
5. 在 **「虚拟内存」** 区域，点击 **「更改」**
6. **取消勾选**「自动管理所有驱动器的分页文件大小」
7. 选择系统盘（通常是C:）
8. 选择 **「自定义大小」**
9. 设置：
   - **初始大小 (MB)**: `16384` （16GB）
   - **最大值 (MB)**: `32768` （32GB）
10. 点击 **「设置」**，然后 **「确定」**
11. **重启电脑**使更改生效

---

### ✅ 解决方案 3：关闭其他占用内存的程序

- 关闭浏览器、编辑器、游戏等
- 打开任务管理器（Ctrl+Shift+Esc）
- 在「性能」→「内存」中查看占用情况
- 结束不必要的进程

---

### ✅ 解决方案 4：始终使用轻量模型和CPU

在前端选择：
- **模型**: ViT-GPT2 图像描述（轻量、稳定）
- **设备**: CPU

这个组合对内存需求最低。

## 技术细节

### 低内存模式做了什么优化？

```python
# 限制线程数减少并发内存占用
os.environ["OMP_NUM_THREADS"] = "2"

# CUDA内存优化
os.environ["PYTORCH_CUDA_ALLOC_CONF"] = "max_split_size_mb:256,expandable_segments:True"

# 默认轻量模型
os.environ["MODEL_NAME"] = "nlpconnect/vit-gpt2-image-captioning"
```
